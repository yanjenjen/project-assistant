# design-sketch｜設計草圖 / PPT 架構 / 流程圖

> 版本：v0.1.0｜狀態：試行中
> 工作區：Claude一般事務（外層通用）

---

## 一、定位

協助規劃視覺架構與草圖設計，包含：

- 規劃 PPT 頁面架構與邏輯層級
- 整理文件視覺層級與資訊架構
- 繪製流程圖草稿（文字描述或 Mermaid 格式）
- 設計 Demo 展示邏輯與說明順序
- 對系統畫面提出呈現與調整建議

**不取代** ERP Cowork 的 `odoo-ui-analyzer`（負責分析既有 Odoo 畫面輸入）；本 Skill 負責產出視覺草圖或呈現建議（輸出），方向相反，不重疊。

---

## 二、觸發條件

**主要觸發詞：**
- `設計草圖：`
- `幫我畫流程圖`
- `幫我做 PPT 架構`
- `Demo 畫面建議`
- `design-sketch：`

**ERP Cowork 內層引用：**
在 ERP Cowork 工作中引用本 Skill 能力時，須使用上述明確前綴。輸出結果僅供參考，不得直接作為 ERP 正式交付文件。

---

## 三、輸出格式

依任務類型輸出，常見格式：

**PPT 架構：**
```
【PPT 架構建議】
主題：
建議頁數：
頁面結構：
  Slide 1：（標題 / 內容重點）
  Slide 2：
  ...
備註：
```

**流程圖（文字描述）：**
```
【流程草圖】
流程名稱：
起點 → 步驟 1 → 步驟 2 → ... → 終點
例外 / 分支：
備註：
```

**Demo 展示邏輯：**
```
【Demo 展示邏輯】
展示目標：
展示順序：
  1.
  2.
  3.
重點說明：
```

## 四、邊界

| Skill / 工具 | 關係 |
|---|---|
| `odoo-ui-analyzer`（ERP Cowork） | 不重疊。`odoo-ui-analyzer` 分析現有 Odoo 畫面（輸入）；本 Skill 產出視覺草圖（輸出） |
| `anthropic-skills:pptx`（平台層） | 互補。本 Skill 規劃架構；pptx Skill 產出實際 .pptx 檔案 |

---

## 五、寫檔限制

預設只在對話中輸出，不自動寫入任何檔案。若需寫入，需另行確認寫入位置與範圍。

---

## 六、回復方式

若觸發條件錯誤或與 `odoo-ui-analyzer` 混淆：

1. 停止使用本 Skill
2. 回報問題，提出停用計畫
3. 等待使用者授權後封存
