## 一、文件目的

本文件依據會議側錄字幕初步整理，重點彙整本次會議中與「資料補齊、文件追繳、後續系統建檔」相關之待辦事項，作為專案後續追蹤與內部作業分工依據。

## 三、會議重點摘要

1. 本次會議主要針對系統畫面與資料匯入／建檔狀況進行確認。
2. 目前經初步比對後，已有部分商品或檔案資料可先行輸入系統。
3. 會中提及目前可正確先輸入者為 4 個檔案／4 個商品。
4. 後續仍需依客戶補件狀況，逐步補齊資料並完成系統建檔或資料維護。
5. 會議錄影／側錄資料可能由對方會議主辦方提供，需確認雙方是否皆有保留會議紀錄或錄影檔。

## 五、建議追蹤欄位

後續若需建立追繳文件清單，建議至少包含下列欄位：

| 欄位 | 說明 |
|---|---|
| 文件編號 | 便於追蹤與對照會議紀錄 |
| 文件名稱 | 例如商品主檔、規格表、申請規則、附件圖檔等 |
| 對應商品／流程 | 標示該文件對應之商品、流程或系統功能 |
| 應提供單位 | 客戶負責部門或窗口 |
| 顧問端負責人 | 內部追蹤窗口 |
| 需求原因 | 說明該文件為系統建檔、欄位比對、流程設定或測試所需 |
| 預計提供日 | 客戶承諾或專案要求日期 |
| 實際收到日 | 實際收到文件日期 |
| 文件版本 | 避免舊版與新版混用 |
| 處理狀態 | 未提供、已收到、待確認、可建檔、已建檔、退回補正 |
| 備註 | 記錄例外狀況、缺漏欄位或待決議事項 |

## 六、待確認事項

1. 本次會議實際日期、出席人員與客戶窗口姓名。
2. 「4 個檔案／4 個商品」的完整清單、檔名、商品名稱與版本。
3. 本週需補交的資料項目、承諾提供人與明確截止日期。
4. 字幕中提及「規格」、「申請規則」等內容之實際業務含義。
5. 會議錄影是否可取得，以便補正目前字幕辨識不清之段落。

## 七、後續建議作業

1. 先建立一份正式「追繳文件清單」，依本紀錄第四節待辦事項拆成可追蹤項目。
2. 向客戶窗口確認本週應提供資料清單與截止日期。
3. 顧問端收到文件後，依「可建檔／待補正／待釐清」三類進行分類。
4. 對已可建檔之 4 個商品或檔案，補上系統建檔狀態與測試結果。
5. 針對語意不清的會議內容，回看錄影或請會議參與者確認後更新本紀錄。
