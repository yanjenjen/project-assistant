## 4. 已確定的處理規則

* 通用內控制度 DOC 可作為通用範本萃取來源，但需逐份萃取、逐份確認。
* 通用內部稽核制度 DOC 可作為稽核程序與查核重點參考，但應在內控制度流程骨架建立後再補充。
* 通用會計制度 DOC 可作為 ERP 會計制度理解來源，但不可直接視為客戶現行制度。
* `好德科技/` 只能作為案例學習，不可納入通用範本萃取。
* `好德科技/` 若後續使用，需另開獨立任務，並完成去識別化。
* ZIP 解壓需要使用者明確授權。
* 解壓前需先檢查目標資料夾是否已存在、是否已有內容。
* 若解壓目標資料夾已存在且已有內容，Cowork 必須停止，不得覆蓋、不合併、不重新解壓。
* 解壓後原始 ZIP 需保留，不可刪除。
* Phase 3A 僅允許解壓與結構確認，不允許讀全文。
* Phase 3B 才能開始內容萃取，且第一份建議為 `內控制度-採購及付款循環.doc`。
* 不可一次萃取 37 份 DOC。
* 不可直接寫入通用流程知識庫。
* 知識庫寫入必須滿足：已脫敏、可跨企業複用、非客戶專屬、使用者確認。
* README / SKILL.md / CLAUDE.md 目前不應再擴寫，除非真實任務暴露明確缺口。
* project-dashboard 暫緩，第二個真實專案啟動時再建立。
* skills-index 暫緩，Skill 數量達 5 個以上或跨 3 類以上再建立。
* Odoo Skills 暫緩搬移，需先驗證分類子資料夾可穩定識別 Skill。
* `.gitkeep` 不應由 Cowork 使用永久刪除權限處理；若要刪，建議使用者手動刪除後再補記。
* 所有改名、搬移、解壓、刪除、寫入知識庫動作都需使用者再次授權。
* 所有重要異動需補記任務包 worklog 與總 worklog。
