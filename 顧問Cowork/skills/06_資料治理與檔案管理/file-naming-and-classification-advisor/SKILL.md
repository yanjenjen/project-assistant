# file-naming-and-classification-advisor｜文件命名與歸檔建議

> 版本：v0.1.0｜狀態：試行中（已完成一次檔名清單測試，尚未完成實際改名 / 搬移測試）
> 預設行為：只產出建議表。改名與搬移須使用者明確授權後，依批次執行。

---

## 使用時機

當使用者提出以下任一需求時，啟用此 Skill：

- 資料夾中有大量未命名或命名混亂的檔案需要整理
- 新收到客戶資料包，需判斷應放入哪個 Cowork 資料夾
- 定期清理 `05_暫存區/` 時，需確認每個檔案的下一步
- 外層資料夾有混入不屬於該分類的檔案
- 確認 `03_公司範例模板/` 現有檔案命名是否符合規則

---

## 不適用情境

- 使用者直接要求「幫我改名」或「幫我搬移」：須先執行本 Skill 產出建議表，確認後才可執行
- 單純對話問答，不涉及資料夾掃描
- 需要讀取並分析檔案**內容**的任務（屬於 `document-template-pattern-extractor` 的工作）
- 使用者只想確認特定一份檔案的命名是否正確（過輕量，直接回答即可）

---

## 標準工作流程

收到整理需求後，**依序執行，不跳步驟**。

### Step 1：確認掃描範圍

詢問或確認以下資訊：
- 要掃描的資料夾路徑或名稱
- 是否只掃描第一層，或需包含子資料夾（**預設只掃描第一層**）
- 若使用者提到的檔案可能屬於特定專案，先確認企業代號或任務包名稱（見「專案代號確認規則」）

### Step 2：掃描並列出檔案清單

使用工具列出指定資料夾的檔案清單（第一層）。

**壓縮檔規則：**
- ZIP / 壓縮檔預設**只依檔名判斷**，不自動解壓、不讀取內部檔案
- 若需解壓或讀取內容，必須取得使用者**明確授權**後才可執行
- 若壓縮檔檔名疑似含敏感資訊，依 A 級規則處理，不得解壓

**停止條件：**
- 若發現疑似 A 級敏感資料夾，停止深入並回報使用者

### Step 3：逐檔判斷並輸出建議表

對每個檔案依序判斷：文件類型、敏感等級、命名模式、建議新檔名、建議歸檔位置、判斷信心、是否可改名、是否可搬移。

### Step 4：確認有無需優先處理的敏感檔案

將 A 級與 B～A 待確認的檔案獨立列出，提示使用者優先確認，**不得進入批次執行**。

### Step 5：依使用者指示進入執行階段

執行階段規則見本文末節。未收到使用者明確授權前，**不得執行任何異動**。

## 標準輸出表格欄位

| 欄位 | 說明 |
|---|---|
| 目前檔名 | 原始檔案名稱（完整，含副檔名） |
| 推測文件類型 | 依檔名與副檔名判斷 |
| 敏感等級 | A / B～A待確認 / B / C / 無 |
| 判斷信心 | 高 / 中 / 低 |
| 命名模式 | 公司範本 / 學習素材 / 制度參考 / 專案資料 |
| 建議新檔名 | 符合命名規則的建議檔名 |
| 建議歸檔位置 | 建議放入哪個資料夾（含路徑） |
| 是否可改名 | ✅ 可執行 / ⏸ 待確認 / ❌ 不建議 |
| 是否可搬移 | ✅ 可執行 / ⏸ 待確認 / ❌ 不建議 |
| 備註 / 待確認 | 無法判斷類型、需補充資訊、特殊注意事項 |

**判斷信心規則：**

| 信心等級 | 判斷標準 |
|---|---|
| 高 | 檔名已明確顯示文件類型與用途，可直接判斷 |
| 中 | 可推測類型，但仍需確認是否為空白範本或真實資料 |
| 低 | 檔名涉及敏感關鍵字、資料庫、連線資訊、客戶資料，或類型完全不明 |

## 專案代號 / 企業代號確認規則

若檔案疑似屬於特定客戶專案，但使用者未提供專案代號、企業代號或任務包名稱：

- **必須先詢問**，不得直接建議歸入 `90_正式交付成果` 或 `91_跨員工協作區` 的任何專案資料夾
- 詢問範例：「這份檔案疑似屬於特定專案，請問對應的企業代號或任務包名稱是？」
- 在使用者確認前，建議歸檔位置填寫「待確認企業代號後決定」

若檔案屬於以下類型，則**不需要**詢問企業代號，可直接依外層資料夾規則建議歸檔：
- 共用範本（`03_公司範例模板/`）
- 學習素材（`02_學習素材/`）
- 制度參考（`04_制度範本參考/`）
- 工作區通用說明文件

## 命名規則

### 公司範本

**優先保留原始分類代碼**（如 A1、B1、B2、B3、C1、D2、E1、E3），以利追溯公司文件分類體系：

```
範本_{原始代碼}_{文件類型}_{用途或來源}_v{版本}.{副檔名}
```

範例：
- `D2_艾創點-ERP輔導備忘錄YYMMDD.docx` → `範本_D2_ERP輔導備忘錄_艾創點_v1.docx`
- `B1_XX公司-(系統名稱)-系統安裝暨交付清單YYMMDD.docx` → `範本_B1_交付清單_系統安裝_v1.docx`
- `A1_XX公司-連線資訊.docx` → `範本_A1_連線資訊表_公司範例_v1.docx`（僅限確認為空白範本後）

若原檔名無分類代碼，使用：
```
範本_{文件類型}_{用途或來源}_v{版本}.{副檔名}
```

### 學習素材

```
學習素材_{主題}_{來源屬性}_{版本或日期}.{副檔名}
```

範例：`學習素材_簡報技巧_外部參考_v2.pdf`

### 制度參考

```
制度參考_{主題}_{來源屬性}_{版本或日期}.{副檔名}
```

範例：`制度參考_內部控制制度_通用_v1.zip`

### 禁止的命名模式（所有類型通用）

- 含空格（改用底線）
- 含 `(1)`、`(2)` 等重複下載流水號（建議移除或改為 `_v2`）
- 含 `最新`、`final`、`new` 等模糊詞
- 僅含日期無說明（如 `20260101.docx`）
- 全英文但無法判斷內容（如 `document1.docx`）

## 歸檔判斷規則

| 文件特徵 | 建議歸檔位置 |
|---|---|
| 客戶提供的原始匯出資料（未整理） | `01_原始資料_不直接處理/` |
| 學習素材、課程講義、行業知識文章 | `02_學習素材/` |
| 公司標準模板、去識別化範例 | `03_公司範例模板/` |
| 外部法規、制度辦法、內控範本 | `04_制度範本參考/` |
| 尚未完成或待確認的草稿 | `05_暫存區/` |
| 正式交付客戶的文件（需確認企業代號） | `顧問Cowork/90_正式交付成果/{企業代號}/` |
| 跨員工協作任務包產出（需確認任務包） | `顧問Cowork/91_跨員工協作區/{任務包}/` |
| A 級敏感資料 | 不建議放入一般資料夾；標示後待使用者決定處置方式 |
| 已過時、已替代的檔案 | `99_封存/` |
| 無法判斷歸屬 | 標示「待確認」，請使用者補充說明 |

## 敏感資料判斷規則

| 等級 | 定義 | 典型範例 | 處理方式 |
|---|---|---|---|
| **A 級** | 若外洩可直接造成系統入侵或資料外洩 | 連線資訊、帳密、VPN 設定、DB 連線字串、系統後台憑證 | 不得寫入知識庫、不得放入一般資料夾；需提示使用者人工處置 |
| **B～A 待確認** | 檔名或類型疑似含敏感資訊，需確認內容 | 資料庫結構、含巨集的 .xlsm、系統架構設計文件 | AI 不讀取內容；先確認是否為空白範本，再判定最終等級 |
| **B 級** | 含客戶機密營運資訊 | 客戶財務資料、合約金額、組織圖、流程規格 | 可保存但須隔離；不寫入共用知識庫 |
| **C 級** | 含客戶名稱或代號，但無機密營運內容 | 含客戶名稱的備忘錄、SOP 草稿 | 正常處理；知識庫沉澱前需去識別化 |
| **無** | 完全通用，無客戶資訊 | 空白制度範本、公司模板、學習素材 | 正常處理 |

**自動觸發規則：**
- 檔名含「帳號、密碼、Token、IP、DB、VPN、憑證、連線」→ 自動標記 A 級預警
- 檔名含「XX公司、客戶、系統名稱、YYMMDD」佔位符 → 標示「待確認是否為空白範本」，確認前不得標為無敏感
- 建議表中若需呈現疑似敏感片段，應遮蔽敏感部分，僅保留可辨識用途的最小資訊

## 執行階段規則

### 第一批：只改名、不搬移

條件（須全部達成）：
1. 建議表已輸出，使用者已閱讀
2. 使用者明確授權「第一批改名」
3. 該批次內所有檔案敏感等級為「無」且已確認為空白範本
4. A 級、B～A 待確認檔案**不得**進入此批次

### 第二批：搬移至其他資料夾

條件（須全部達成）：
1. 第一批改名已完成（或使用者明確跳過）
2. 搬移計畫已逐筆列出原路徑 → 目標路徑
3. 使用者逐批確認目標位置
4. 涉及多企業時，已確認每筆異動的企業代號

### 執行前七項必要條件（缺一不可）

1. 建議表已輸出，使用者已閱讀
2. 使用者明確說「確認執行」或指定批次
3. A 級及 B～A 待確認檔案已排除在執行範圍外
4. 涉及多企業時，已確認每筆異動的企業代號
5. 改名或搬移計畫已逐筆列出（原路徑 → 目標路徑），使用者確認無誤
6. 改名與搬移已分批規劃，不混合執行
7. 壓縮檔若需解壓，已取得使用者獨立授權

### 回復原則

若改名或搬移後發現錯誤，立即回復原狀並回報使用者，不得自行判斷後繼續執行。

---

## 禁止事項

- 禁止未取得使用者確認前執行任何改名、搬移、刪除動作
- 禁止讀取 A 級或 B～A 待確認等級的檔案內容
- 禁止自動解壓 ZIP / 壓縮檔（需使用者明確授權）
- 禁止將含客戶資訊的路徑或內容寫入 Cowork 共用知識庫
- 禁止自行判斷「舊版本可以刪除」並執行刪除
- 禁止將 A 企業的檔案建議歸入 B 企業的資料夾
- 禁止在未確認企業代號前，建議歸入 90 或 91 的任何專案資料夾
- 禁止在建議表以外的位置輸出含敏感資訊的檔案路徑
