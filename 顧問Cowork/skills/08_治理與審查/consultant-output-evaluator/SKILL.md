# consultant-output-evaluator｜顧問成果品質評估

> 版本：v0.1.0｜狀態：試行中
> 主要用途：ERP 顧問工作成果品質評估，不等同正式驗收，評估結論為建議。

---

## 一、Skill 定位

ERP 顧問工作成果、文件、Skill 輸出的品質評估工具，職責如下：

- 評估輸出的**完整性、正確性、可追蹤性、風險揭露、是否過度推測、是否符合既有規則 / 範本**
- **只能在使用者明確要求「評估 / 驗收 / 審查」時啟動**，不得自動介入任何任務流程
- **不得自行判定文件已正式通過**，只能提供評估建議；正式驗收仍需使用者或主管確認
- 評估結論為建議，不取代人工判斷

---

## 二、使用情境

啟用本 Skill 的情境：

- 評估**會議紀錄**是否完整（決議、Action Item、格式符合性）
- 評估**規格書**是否符合撰寫規則、ERP 對應是否準確
- 評估**制度萃取草稿**格式一致性、控制點分類完整性、待確認事項是否齊全
- 評估 **Skill 輸出**是否在授權範圍內、是否過度推測、輸出格式是否符合 SKILL.md
- 評估**任務計畫**的分工、風險揭露、前提條件、依賴關係是否清楚
- 確認是否有缺漏、過度推測、格式不一致、風險未揭露

---

## 三、觸發條件

**主要觸發詞**（推薦使用，意圖最明確）：
- `evaluate：`
- `評估一下：`
- `幫我驗收：`
- `審查這份：`
- `幫我檢查一下：`

**輔助觸發詞**（語境明確為品質審查時啟用）：
- `這份文件有沒有問題`
- `這個輸出可以用嗎`
- `幫我確認這個符不符合規則`

**輔助觸發詞判斷原則**（須同時滿足以下三項才啟動）：
1. 使用者語境明確是在要求品質審查、驗收或評估
2. 對話中已有明確的評估對象，例如文件草稿、Skill 輸出、任務計畫、制度萃取草稿或會議紀錄
3. 不得因一般提問、普通討論、單純詢問意見而自動啟動

**觸發限制：**
- **僅在使用者明確要求品質評估 / 驗收 / 審查時啟動**
- 不得因一般提問、回顧、討論而自動啟動
- 不得在使用者未主動呼叫的情況下主動建議「要不要評估一下」並自行執行

---

## 四、不適用情境

以下情境**不應使用本 Skill**：

- **自動審查**：本 Skill 不得在未被明確呼叫的情況下介入任何任務
- **正式驗收**：本 Skill 輸出為建議，不取代使用者或主管的正式驗收決定
- **即時單筆記錄**：請使用 `quick-capture`
- **段落回顧**：請使用 `session-reflect`（self-review 視角，與本 Skill 的 quality-check 視角互補）
- **任務規劃**：請使用 `erp-task-planning-assistant`
- **跨企業評估標準套用**：不得將 A 企業的評估標準直接套用至 B 企業輸出，除非使用者明確指定為通用規則

---

## 五、可評估對象

| 對象 | 評估重點 | 參考規則 / 範本 |
|---|---|---|
| 會議紀錄 | 完整性（決議、Action Item）、格式符合性、可追蹤性 | WR-002 會議紀錄撰寫規則 |
| 規格書 | 完整性、正確性、欄位格式、ERP 對應準確度 | WR-001 規格書撰寫規則 |
| 制度萃取草稿 | 格式一致性、控制點分類（TC-007）、待確認事項完整性（TC-006/008） | REF 任務包萃取格式 |
| Skill 輸出 | 是否過度推測、是否在授權範圍內、輸出格式是否符合 SKILL.md | 對應 SKILL.md |
| 任務計畫 | 分工清楚、風險揭露、前提條件、依賴關係 | CLAUDE.md 任務執行規則 |

---

## 六、評估維度

| 維度 | 定義 | 不通過的典型情境 |
|---|---|---|
| 完整性 | 必要欄位 / 章節 / 待確認項目是否齊全 | 規格書缺少 ERP 對應欄位；會議紀錄無 Action Item |
| 正確性 | 內容是否與已知事實、規則、授權結果一致 | 制度萃取草稿引用錯誤企業的流程資料 |
| 可追蹤性 | 版本標示、企業代號、負責角色、日期是否清楚 | 文件缺少版本號或企業代號 |
| 風險揭露 | 風險點、阻塞點、待確認事項是否已明確標示 | 有待確認事項但未加入 99_待確認事項.md |
| 是否過度推測 | 是否有未標示「待補資料」的假設性內容 | 制度萃取草稿將「可能的控制點」寫成定論 |
| 符合既有規則 / 範本 | 是否依循工作區規則、撰寫規則或 Skill 定義輸出 | 文件未依 WR-001 格式；Skill 輸出超出 SKILL.md 授權範圍 |

## 七、與既有 Skill 的邊界

| Skill | 職責 | 與本 Skill 的關係 |
|---|---|---|
| `document-template-pattern-extractor` | 從範本萃取撰寫規則，建立評估基準 | 不重疊。extractor 建立標準；evaluator 使用標準評估。可串接 |
| `erp-task-planning-assistant` | 產出任務計畫 | 不重疊。planning 產出計畫；evaluator 評估計畫品質 |
| `quick-capture` | 工作中即時單筆記錄 | 完全不重疊。capture 是輸入工具；evaluator 是輸出審查工具 |
| `session-reflect` | 段落結束後的自我回顧（self-review） | 互補。reflect 是作者視角；evaluator 是品質檢查視角 |
| `odoo-ui-analyzer` | Odoo 畫面辨識與顧問分析 | 不重疊。ui-analyzer 產出分析報告；evaluator 可評估該報告品質 |
| `consultant-output-evaluator`（本 Skill） | 評估上述成果品質，不取代原 Skill | — |

## 九、寫檔限制

- **預設只在對話中輸出**，不自動寫入任何檔案
- 不自動寫入 worklog、任務包、正式文件或 `90_正式交付成果`

## 十、回復方式

若安裝後觸發條件出現錯誤、被誤用為自動審查，或與其他 Skill 觸發混淆：

1. 立即停止使用本 Skill
2. 向使用者回報問題描述（誤觸情境、混淆的 Skill 名稱）
3. 提出停用 / 封存計畫，等待使用者授權
4. 授權後將 `skills/08_治理與審查/consultant-output-evaluator/` 移至 `skills/99_封存/`
5. 若 `08_治理與審查/` 內已無其他 Skill，一併評估是否移除空資料夾（需另行授權）
6. 同步更新 `skills/README.md` 索引與 `skills/第一階段Skill導入計畫_v0.1.md` 狀態
