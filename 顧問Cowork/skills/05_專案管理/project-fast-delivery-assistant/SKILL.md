# project-fast-delivery-assistant｜專案趕工交付助理

> 版本：v0.2.0｜狀態：試行中
> 主要用途：在任務卡住、範圍過大或需要衝刺時，快速收斂今日可執行的最小交付方案。
> 升版說明：v0.1.0 → v0.2.0 新增任務類型判斷、P0/P1/P2 分類、Skill 選用建議、
> 今日不做清單，調整觸發詞。

---

## 使用時機

當使用者遇到以下任一情境時，啟用此 Skill：

- 任務很多，不確定今天應該先完成哪一項
- 任務範圍太大，需要拆成最小可交付成果
- 已有工作區資料，但缺少下一步執行順序
- 需要在不完整資料下先產出可檢查、可確認、可交接的草稿
- 任務包進度停在待確認，需要整理「確認後可立即執行」的方案
- 使用者要求「不要再規劃太大，先做能落地的」

---

## 不適用情境

- 使用者已明確指定要執行單一操作，例如「幫我改名這 3 個檔案」
- 任務涉及 A 級敏感資料，且尚未完成安全處置確認
- 任務需要法律、會計、資安或合約判斷，但缺少必要資料
- 使用者要求正式交付成果，但尚未確認企業代號、任務範圍與資料來源

---

## 標準工作流程

收到趕工需求後，依序執行以下步驟，**不跳步驟**。

### Step 1：判斷任務類型

先確認本次任務屬於下列哪一類（可複選）：

| 類型 | 說明 |
|------|------|
| 資料進場 | 新資料放入工作區、命名歸檔、敏感等級初判 |
| 範本萃取 | 從 DOC / PDF / PPT 萃取制度規則、流程、控制點 |
| 需求分析 | 客戶訪談整理、需求缺口分析、功能範圍確認 |
| 規格草稿 | 產出標準作業規格書、API 規格書或技術說明文件 |
| 會議紀錄 | 整理會議內容、決議事項、待確認項目 |
| SOP | 建立或補強標準作業流程文件 |
| 追資料 / 發信 | 整理待確認事項、草擬回覆或追蹤信件 |
| 內控 / 流程整理 | 制度文件整理、控制點分析、簽核流程梳理 |

任務類型影響後續 Skill 選用與風險提示。

### Step 2：判斷今日最小可交付成果

用以下標準選出最小交付方案：

1. 不需要未授權的改名、搬移、刪除或解壓操作
2. 不寫入 `90_正式交付成果`
3. 可在任務包或暫存區留下可追蹤成果
4. 使用者看完後能做出下一個明確決策
5. 可由現有 Cowork 角色或既有 Skill 接手

### Step 3：判斷是否啟用既有 Skill

依任務類型，建議搭配以下 Skill：

| Skill | 適用情境 |
|-------|----------|
| `file-naming-and-classification-advisor` | 資料進場、命名歸檔、敏感等級判斷 |
| `document-template-pattern-extractor` | 範本萃取、文件撰寫規則建立 |
| `odoo-module-spec-writer` | Odoo 模組說明書、功能規格草稿 |
| `odoo-large-feature-dev` | Odoo 大型需求拆解、開發規劃 |

若任務不需要上述 Skill，直接進入 Step 4。

### Step 4：拆出 P0 / P1 / P2

| 優先級 | 定義 | 今日動作 |
|--------|------|----------|
| P0 | 今日必須完成，否則後續任務無法推進 | 立即執行 |
| P1 | 今日可完成，完成後可加速進度 | P0 完成後再做 |
| P2 | 重要但不急，可延後確認或執行 | 記入待確認，今日暫緩 |

每個 P 層最多列 3 項，避免清單過長。

### Step 5：明確列出今日不做事項

每次輸出必須有「今日不做」清單，列出以下類型：

- 範圍太大、需要另外授權的操作
- 可能造成資料混淆或多企業污染的事項
- 與本次任務無直接關聯的延伸工作
- 使用者尚未確認的前提條件

### Step 6：產出可直接執行的工作清單

輸出 3 到 7 項，每項格式：

```
[ ] 動作描述（動詞 + 對象）
    ↳ 預期產出：
    ↳ 放置位置：
    ↳ 依賴條件：（若無則省略）
```

### Step 7：若需檔案操作，先提出操作計畫

凡涉及建立、修改、刪除、搬移、重新命名檔案，**先提出計畫，等使用者確認後再執行**。
不得以「趕工需求」為由跳過操作確認。

---

## 輸出格式

```text
# 今日快速交付建議

## 任務類型
## P0｜今日必做
## P1｜今日可做
## P2｜今日暫緩
## 今日不做
## 可用 Skill
## 建議執行清單
## 待確認事項
```

---

## 禁止事項

- 不新增大型架構或批量建立空資料夾
- 不擴寫 README / CLAUDE.md
- 不寫入知識庫，除非使用者明確授權
- 不自動搬移正式成果到 `90_正式交付成果`
- 不處理未授權敏感資料
- 不自行啟動新任務包
- 不擴大任務範圍
- 不取代使用者決策
- 不以「趕工」為由跳過企業代號確認
- 不在未授權前解壓 ZIP、改名、搬移或刪除檔案
- 不將 B 級或待確認資料寫入共用知識庫
- 不把草稿包裝成正式交付成果
