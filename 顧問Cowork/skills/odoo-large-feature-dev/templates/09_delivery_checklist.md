# 階段 9：交付清單

## 必交付物

### 1. 程式（兩處同步）
- [ ] 當日日期資料夾：`D:\claude\XXX\YYYYMMDD\yc_xxx\`
- [ ] 最新版本：`D:\claude\XXX\最新版本\yc_xxx\`

### 2. 測試流程文件 .docx
含：
- [ ] 文件目的與範圍
- [ ] 模組功能概述
- [ ] 測試環境前置（含 location/partner/picking type 清單）
- [ ] 模組設定（一般設定/校區設定/公式）
- [ ] 商品與歷史銷售準備
- [ ] 15+ 個功能測試案例（TC-XX）
- [ ] E2E 端對端情境
- [ ] 邊界與例外測試（EX-XX）
- [ ] 故障排除（常見錯誤對照表）
- [ ] 驗收簽核欄

### 3. 測試匯入檔
- [ ] 商品主檔 xlsx（分 sheet：學生/教師/無尺寸）
- [ ] 歷史資料 csv（符合 wizard 規格）
- [ ] 測試樣本 xlsx（校區設定、廠商、雙向 partner、需求單樣本、庫存初始化）
- [ ] 目錄 sheet（含建議匯入順序）

### 4. 模擬測試腳本
- [ ] test_simulation.py（單元測試）
- [ ] test_e2e_flow.py（E2E 流程）
- [ ] 測試結果報告（全通過）

### 5. 文件最後必標
- [ ] 「程式正確 ≠ 環境能跑」風險清單
- [ ] 已知 Odoo 環境差異（如 picked 欄位版本依賴）
- [ ] 第一次測試的建議順序

## 同步指令模板

```bash
# 模組同步
rm -rf 最新版本/yc_xxx
cp -r YYYYMMDD/yc_xxx 最新版本/yc_xxx

# 文件同步
cp YYYYMMDD/yc_xxx_測試流程文件.docx 最新版本/

# 測試匯入檔同步
rm -rf 最新版本/測試匯入檔案
cp -r YYYYMMDD/測試匯入檔案 最新版本/
```

## 給使用者的最後訊息範本

```
模組程式：D:\...\yc_xxx
測試文件：D:\...\yc_xxx_測試流程文件.docx
匯入檔：D:\...\測試匯入檔案
測試腳本：outputs\yc_xxx_test\

驗證結果：
- 單元測試 XX 點全通過
- E2E 流程 XX 點全通過
- 修了 X 個 bug

依測試流程文件 3.x 前置 → TC-01 ~ TC-15 → E2E 走一遍。
報錯貼上來再 debug。
```

## 不可

- 沒做模擬測試就交付（沒有環境也要驗）
- 沒列「已知限制」（讓使用者以為什麼都能 100% 跑）
- 文件用簡體中文
- 文件過度書面化（適合上台講的口語才是好文件）
