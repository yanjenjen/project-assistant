# 階段 1：需求釐清問題清單範本

## 用法
收到需求後，先用 AskUserQuestion 工具問完這些問題，再進階段 2。

## 通用問題清單（依需求類型勾選）

### A. 角色與單位
- [ ] 涉及哪些角色？（如：總部、各分公司、客戶、供應商、家長）
- [ ] 是否多公司（res.company）架構？
- [ ] 各角色的數量級？（1 對 N、N 對 N？）

### B. 單據與資料流
- [ ] 共有幾種單據？
- [ ] 每張單據誰建立、誰確認、誰結案？
- [ ] 單據之間的關聯（誰來自誰、誰彙整誰）？

### C. 主檔
- [ ] 商品/服務統一主檔還是各單位獨立？
- [ ] 客戶/供應商建模方式？
- [ ] 是否有歷史資料要帶入？

### D. 庫存（如有）
- [ ] 商品所有權歸屬？
- [ ] 庫存實體位置？
- [ ] 是否需要追蹤批次/序號？

### E. 會計（如有）
- [ ] 跨公司結算方式？
- [ ] 認列營收的時點？
- [ ] 是否雙重認列風險？

### F. 異常
- [ ] 追加/修改的處理方式？
- [ ] 取消的處理方式？
- [ ] 超量/不足的處理方式？

## 寫法範例

收到「集團團購流程」後，我會這樣問：

```
1. 「家長」是 1 個虛擬客戶、還是每個家庭一筆 partner？
2. 月底開的銷售訂單，是「全集團 1 張」、還是「每校 1 張」？
3. 各校對總部的待收款，要用哪種單據呈現？
   (a) Odoo 內部 invoice
   (b) 自訂應付總部單據
   (c) 直接用會計傳票
4. 各校的「總部專屬倉」要怎麼設？
   (a) 倉庫歸屬總部 company、location 用各校編碼區分
   (b) 倉庫歸屬各校 company，但庫存科目掛在總部
5. 品項是統一主檔、還是各校自建？
```

## 不可

- 不要自己想答案
- 不要問太多次（一次 5~6 題收斂）
- 不要超過 4 個選項（AskUserQuestion 限制）
