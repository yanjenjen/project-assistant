# 階段 4：細節再次收斂

## 用法
階段 3 挖出的細節，逐項問清楚「要做進系統 or 留人工」。

## 典型細節議題

### 計算公式
- 固定 1 種 / 多種類型 / 自由表達式？
- 推薦做法：3~4 個預設類型 + 1 個 custom 自由表達式

### 歷史資料
- 從系統 SO 反查 / CSV 匯入 / 兩者皆要？
- 第一年用：必須支援匯入

### 中介流程（如：跨單位調撥）
- 與主單耦合 / 獨立 model？
- 推薦：獨立 model + 主單上有快捷入口

### 異常處理（如：批發補量、總部備用）
- 系統自動分攤 / 留人工填欄位？
- 推薦：留人工欄位，因為規則每次不同

### 追加變更
- 新單據 / 改既有單據？
- 推薦：新單據 + is_supplement 標記

### 角色/類別分流
- 欄位標記 / 獨立 model？
- 推薦：欄位標記 + view domain 過濾

## 範例：總部代辦品的細節決定

| 議題 | 決定 |
|---|---|
| 公式 | 4 預設類型 + 自由表達式 |
| 歷史銷售 | 反查 + 匯入皆做 |
| 跨校調撥 | 獨立 model + 彙整單快捷入口 |
| 批發補量 / 總部備用 | 留人工欄位（qty_order_manual / qty_hq_reserve） |
| 追加 | 新單據（is_supplement=True） |
| 教師/學生分流 | product 上 gp_user_type + 單據 user_type |
