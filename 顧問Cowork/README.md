# 1. 工作區簡介
`顧問Cowork` 是專為 ERP 顧問 AI 小員工設計的工作區。此資料夾內的內容用於協作、任務執行、交付整理與正式成果保存。它不是一般聊天空間，而是專注於 ERP 顧問專案、流程知識與交付文件的正式工作場域。

# 2. 工作區使用原則
- 使用提醒：此工作區應以 ERP 顧問專案為中心，避免放入與專案無關的資料。
- 使用提醒：所有檔案操作前，建議先說明目的與預期結果，並確認是否需要移動或修改。
- 使用提醒：遇到資訊不完整時，請標註「待補資料」或「需確認」，避免直接補齊假設內容。
- 使用提醒：敏感資料不要放在不必要的檔案、說明文件或協作草稿中。

# 3. 三位 AI 員工簡介
- `員工_01_專案分析師`
  - 主要負責：依據企業流程知識庫進行需求與缺口分析，判斷專案範圍與風險。
- `員工_02_文件資料助理`
  - 主要負責：依據分析結果製作正式文件、資料表、匯入模板與交付資料。
- `員工_03_流程內控助理`
  - 主要負責：建立與維護企業專屬流程知識庫，提出流程風險、例外與系統對應資訊。

# 5. 各資料夾用途說明
- `00_共用規則與知識庫`
  - 存放工作區共用規則、流程知識庫、標準模板與核心參考資料。其中 `01_工作區規則/多企業專案資料隔離規則.md` 為多企業 / 多專案並行時的資料隔離完整規則，所有任務執行前應優先參考。
- `90_正式交付成果`
  - 存放可交付的正式文件、報告、SOP、資料表和最終成果。應依企業 / 專案建立獨立子資料夾（格式：`[企業代號]/`），僅放已由使用者確認的正式版本，不得存放草稿或未確認文件。
- `91_跨員工協作區`
  - 存放草稿、交接紀錄、協作中的中間成果與待確認項目。為草稿 / 協作 / 中間成果區，每個任務應依「`[企業代號]_[YYYYMMDD]_[任務名稱]`」建立獨立任務包資料夾，不同企業的任務包嚴格分隔。
- `skills`
  - 存放支援性資源、工作區工具說明、技巧說明檔案。
- `員工_*` 資料夾
  - 作為三位 AI 員工各自的工作領域，用於保存角色專屬的分析、文件、知識或協作材料。

# 6. 新任務進來時的使用流程
0. 確認本次任務所屬企業或專案名稱：明確指定企業代號或專案名稱。未指定前，不得在 `90_正式交付成果` 或 `91_跨員工協作區` 建立任何資料夾、任務包或檔案。詳見 `00_共用規則與知識庫/01_工作區規則/多企業專案資料隔離規則.md`。
1. 確認任務屬性：流程分析、需求規劃、文件交付或其他。
2. 判斷主責角色：視任務重點選擇員工_03、員工_01 或員工_02。
3. 收集現有資料與需求，標註待補或需確認項目。
4. 若需跨角色協作，將已完成內容與待確認事項清楚交接。
5. 任務完成後，將正式成果整理至 `90_正式交付成果`，協作草稿保留於 `91_跨員工協作區`。

# 7. 文件與資料放置規則
- 正式成果應放在 `90_正式交付成果`。
- 協作中的草稿、分析初稿與交接資料應放在 `91_跨員工協作區`。
- `00_共用規則與知識庫` 保留核心規則與企業流程知識庫，不作為臨時草稿區。
- 文件應標示版本、狀態、負責角色與必要的待確認事項。
- 非正式內容不應直接標示為正式版。
- `90_正式交付成果` 應依企業 / 專案建立子資料夾，不同企業的成果不得混放。
- `91_跨員工協作區` 的任務包資料夾命名應包含企業代號，格式為 `[企業代號]_[YYYYMMDD]_[任務名稱]`。
- 未指定企業或專案名稱時，不得建立或寫入 `90_正式交付成果` 與 `91_跨員工協作區` 的任何資料夾、任務包或檔案。
- 工作區設有兩個層級的 worklog：`顧問Cowork/worklog.md` 記錄工作區層級的建置、規則、模板、Skill、人員設定與治理變更；任務包內的 `worklog.md` 記錄單一任務包的執行過程、角色交接與交付狀態。當任務包開始填寫 01 / 02 / 03 / 99 或進入員工交接時，應在任務包內建立專屬 worklog.md，不得將任務執行細節長期寫入總 worklog。

# 8. 外層資料庫與顧問Cowork的分工
- 外層 `艾創點數位-ERP顧問`
  - 是 ERP 顧問總資料庫。
  - 適合放：原始資料、學習素材、公司範例模板、制度範本參考、暫存資料。
- 內層 `顧問Cowork`
  - 是 AI 小員工正式工作區。
  - 適合放：AI 工作規則、三位員工工作區、跨員工協作草稿、正式交付成果、`skills`。

# 9. 不應放入本工作區的資料
- 不應放：帳密、IP、API Key、客戶個資、合約內容、報價、內部成本等敏感資訊。
- 不應放：與 ERP 顧問專案無關的長期資料或個人文件。
- 不應放：未經確認的假設內容，除非標註為「待補資料」或「需確認」。

# 10. 後續維護方式
- 定期檢視 `CLAUDE.md` 與 `README.md` 是否仍符合工作實際需求。
- `流程內控助理` 可持續更新企業流程知識庫。
- 任務執行中如發現規則或結構問題，可提出優化建議並納入後續更新。
- 保留工作紀錄與版本資訊，方便後續追蹤與維護。

# 11. 多企業 / 多專案防混淆規則

本工作區支援同時服務多家企業或多個 ERP 導入專案。為防止不同企業資料互相混入，工作區訂有完整的多企業防混淆規則，完整版本存放於：

`00_共用規則與知識庫/01_工作區規則/多企業專案資料隔離規則.md`

核心要點如下：

- 每次任務開始前，必須明確指定企業名稱或專案代號
- 未指定企業時，不得建立或寫入 `90_正式交付成果` 與 `91_跨員工協作區` 的資料夾、任務包或檔案
- 不得將 A 企業資料寫入 B 企業資料夾
- `91_跨員工協作區` 為草稿與協作中間成果區，依企業代號與任務日期建立任務包
- `90_正式交付成果` 為已確認可交付版本區，依企業 / 專案分類
- 所有正式產出檔名應包含企業代號作為首碼
