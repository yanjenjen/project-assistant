## 3. 銷售 / 收款流程節點

```text
CS101 銷售預測
  → CS102 銷售計劃
      ↓
CS103 訂單處理（訂單 / 詢價 / 報價 / 樣品與成本分析）
      ↓
CS104 徵信（客戶資料 / 信用額度 / 擔保或現金付款判斷）
      ↓
CS105 營業員管理（目標 / 日報 / 拜訪 / 業績分析）
      ↓
CS106 交貨（交期管理 / 出庫 / 交運 / 外銷裝船）
      ↓
CS107 發票開立與作廢
      ↓
CS108 應收帳款（入帳 / 明細表 / 帳齡 / 呆帳 / 沖銷）
      ↓
CS109 客訴處理、CS110 售後服務
      ↓
CS111 簽證押匯、CS112 收款
      ↓
CS113 銷貨折讓
      ↓
CS114 差異分析（收入 / 成本 / 毛利 / 費用 / 佣金 / 運費）
```

## 7. ERP 顧問訪談問題

### 銷售預測 / 計劃
1. 銷售預測目前由哪些單位提供資料？是否有固定版本與核准流程？
2. 銷售目標、預算與實際業績目前如何比較？系統是否可按業務、客戶、產品別分析？
3. 銷售計劃修正是否有核准流程？是否保留修正原因與版本紀錄？

### 訂單 / 授信
4. 訂單目前由誰建立與核准？訂單變更、取消、作廢是否有留存紀錄？
5. 報價單與訂單之間是否需要系統串接？售價權限是否依角色或金額控管？
6. 客戶授信額度由哪個單位維護？超額訂單是否需要系統阻擋或特案核准？
7. 新客戶建檔是否需要徵信資料、擔保資料或付款條件審核？

### 出貨 / 發票 / 應收
8. 出貨單、交運單、發票目前是否由同一系統產生？三者是否可勾稽？
9. 發票作廢與重開是否需要審核？是否需控管發票連號與作廢原因？
10. 應收帳款明細表與帳齡分析目前由哪個系統產出？逾期帳款是否有提醒？
11. 收款報告單、沖帳與應收帳款是否需要系統自動配對？

### 客訴 / 售後 / 折讓
12. 客訴案件是否有固定編號、責任判定與處理進度追蹤？
13. 退貨、理賠、折讓是否與原訂單、出貨單、發票、應收帳款連動？
14. 保證卡與服務卡是否需要納入 ERP 或另由客服 / 維修系統管理？

### 外銷 / 收款 / 差異分析
15. 外銷信用狀、押匯、報關與裝船文件目前如何歸檔與追蹤？
16. 應收票據、更換 / 延兌、退票是否需系統管理與警示？
17. 銷售差異分析是否需要按產品、客戶、業務、地區、交易條件拆分？
18. 運費、保險費、佣金等直接銷售費用是否需歸屬至訂單或產品別？
