## 使用說明

1. 任一角色發現待確認事項時，直接新增一列至「待確認項目表」。
2. 解決後將該列移至「已解決項目歸檔區」，並填寫解決方式與日期。
3. 標示規則：資料不足請標示 `待補資料`；需客戶或顧問確認請標示 `需確認`。
4. 影響程度說明：高 = 影響上線或主流程；中 = 影響細節或規格；低 = 可延後釐清。
5. **必須進入 99 的項目類型**：凡影響主流程設計、系統設定、簽核規則、單據流、正式交付文件或上線節點的未確認事項，均必須彙整至本清單。
6. 欄位說明：確認對象 / 確認方式 / 預計釐清期限 / 確認結果 / 後續動作 / 影響檔案。
