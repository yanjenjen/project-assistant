## 3. 生產流程節點

| 流程節點 | 作業摘要 | 後續 ERP 訪談關注 |
|---|---|---|
| 製程規劃 | 分析產品、物料規格、零件自製或外包、加工裝配程序、人力設備需求、經濟生產批量 | 是否有標準製程、BOM、工時、機台與批量資料 |
| 負荷規劃 | 測定生產能量，分析人力、機器與工作負荷，安排加班或交期調整 | 是否有產能負荷計算、排程與加班管理資料 |
| 託外加工 | 判斷託工有利性，選定協力廠商，簽訂加工合約，管控供料、交期與品質 | 是否有委外加工、供料、驗收、餘廢料與付款流程 |
| 排程及生產 | 建立基準日程、總生產日程、製造日程與小日程，發佈製造命令 | 是否有製令 / 工單、派工、日報與進度回報機制 |
| 倉儲管理 | 物料編號、驗收、入庫、領料、退料、存量管制與盤點 | 是否有庫存、領退料、批號、先進先出、安全存量控制 |
| 生產管制 | 記錄每日完成數量，比較預計與實際進度，分析停工待料、設備故障、不良率等差異 | 是否有生產進度追蹤、異常回報與補救措施 |
| 品質管制 | 從原物料、製程、在製品到成品進行檢驗、抽驗與異常改善 | 是否有品質檢驗、不良品、重工、退料與改善紀錄 |
| 保養維修 | 建立設備保養卡，制定保養標準與年度 / 月份檢修保養計劃 | 是否有設備保養、故障請修、維修履歷與保養成本資料 |
| 安全衛生 | 依安全衛生法令執行安全裝備、環境衛生、污染防治與教育訓練 | 是否需與 EHS 或設備安全管理流程串接 |
| 生產成本控制 | 計算直接材料、直接人工、製造費用、約當產量與費用分攤 | 是否有成本歸集、工時、材料耗用與製造費用分攤規則 |
| 差異分析 | 分析材料、人工、製造費用、單位工時成本與損益兩平資料 | 是否有標準成本、實際成本、差異原因與責任歸屬 |

## 5. 角色 / 權責 / 簽核點

| 角色 / 單位 | 權責摘要 | 待確認事項 |
|---|---|---|
| 財會單位 | 文件權責單位；參與付款、入帳、成本計算與差異分析 | 實際制度維護與成本資料來源需確認 |
| 生產單位 | 製程規劃、提出託外加工申請、執行生產與品質改善 | 生產單位與生管單位分工需確認 |
| 生管單位 | 負荷規劃、日程安排、加班計劃、進度管制 | 是否有獨立生管角色需確認 |
| 倉儲單位 | 收料、入庫、領料、退料、進出庫紀錄、存量管制與盤點 | 倉儲與物料管理角色需確認 |
| 品管單位 | 原物料、製程、在製品、成品檢驗與不良原因分析 | 品檢站、巡迴抽檢與複檢流程需確認 |
| 保養單位 | 建立保養卡、保養標準、保養計劃與故障請修處理 | 保養分級與委託修造流程需確認 |
| 營業單位 | 負荷不足或無法如期交貨時與客戶協商交期 | 與銷售收款循環交期變更銜接需確認 |
| 現場主管 / 領班 | 派工、加班安排、操作監督、異常回報 | 現場主管核准權限需確認 |
| 協力廠商 | 執行託外加工並配合品質、交期、庫存與餘廢料管制 | 協力廠商管理流程需確認 |
| 總經理 / 主管 | 文件核准、重大工程 / 生產計劃 / 加班等核准 | 核准層級與金額門檻需確認 |

## 6. 主要控制點

### 製程規劃

| 控制點 | 控制類型 | 系統確認狀態 |
|---|---|---|
| 原料及人工使用須經濟合理，製程規劃需考量人力、設備、工具、作業時間與經濟批量 | 權責控制 | 待系統確認 |
| 工作方法、機器設備與工具均應有具體規劃 | 單據控制 | 待系統確認 |
| 製程規劃如有缺失，應即檢討修正 | 例外控制 | 待系統確認 |
| 應經常查核工作人員方法與機器使用是否依製程設計進行 | 系統控制 | 待系統確認 |

### 負荷規劃

| 控制點 | 控制類型 | 系統確認狀態 |
|---|---|---|
| 應測定現有工作能量，以規劃補充人力、機器設備與資金來源 | 系統控制 | 待系統確認 |
| 加班工時與進度應列入生產排程考慮項目 | 系統控制 | 待系統確認 |
| 加班需經核准後通知現場主管安排 | 權責控制 | 待系統確認 |
| 實際加班情形應查核是否有故意延遲或工時記錄不實 | 例外控制 | 待系統確認 |

### 託外加工

| 控制點 | 控制類型 | 系統確認狀態 |
|---|---|---|
| 託外加工須經比較託工與自製成本，確認託工有利 | 權責控制 | 待系統確認 |
| 協力廠商應調查設備能量、財務狀況、品質及交期等資料 | 單據控制 | 待系統確認 |
| 託外加工決價後應與廠商訂立加工合約，並敘明品質標準與收料規定 | 單據控制 | 待系統確認 |
| 應不定期至加工廠商抽檢品質或盤點，並記錄檢驗與稽核結果 | 系統控制 | 待系統確認 |
| 品質不良、超過允許損耗率或餘廢料未繳回時，應依合約扣款或賠償 | 例外控制 | 待系統確認 |

### 排程及生產

| 控制點 | 控制類型 | 系統確認狀態 |
|---|---|---|
| 排定日程時應考慮動作時間研究、製程順序與寬放時間 | 系統控制 | 待系統確認 |
| 應以製造命令發佈生產活動，妥善安排用料、人工與作業程序 | 單據控制 | 待系統確認 |
| 生產進度與成本必須詳加記錄 | 單據控制 | 待系統確認 |
| 對突發狀況應有妥善處理辦法 | 例外控制 | 待系統確認 |

### 倉儲管理

| 控制點 | 控制類型 | 系統確認狀態 |
|---|---|---|
| 原物料與製成品均應編號，分類編號原則應簡單、完整及單一 | 系統控制 | 待系統確認 |
| 到貨驗收應檢核名稱、數量及品質是否與原訂單相符 | 單據控制 | 待系統確認 |
| 物品入庫與出庫均應辦妥手續，並保存連續編號、空白及作廢單據 | 單據控制 | 待系統確認 |
| 領料須經主管核准後倉儲方得發料，並登錄帳冊 | 權責控制 | 待系統確認 |
| 領料量異常、退料、廢料、呆廢料標售與大量盤盈盤虧應追查原因 | 例外控制 | 待系統確認 |
| 安全存量、訂購點、請購點與經濟採購量應依資料分析設定 | 系統控制 | 待系統確認 |

### 生產管制

| 控制點 | 控制類型 | 系統確認狀態 |
|---|---|---|
| 每日完成數量與累積完成數量應記錄並與預定生產量比較 | 系統控制 | 待系統確認 |
| 實際生產與計劃若有差異，應追查原因並擬定處理對策 | 例外控制 | 待系統確認 |
| 生產異常時應立即反應，並採取安全存量、外包、加班等補救措施 | 例外控制 | 待系統確認 |
| 各製程操作標準、作業方法與時間若已建立，應確實執行並適時修正 | 權責控制 | 待系統確認 |
| 應檢討機器與人員工作負荷是否過多、過少或不均 | 系統控制 | 待系統確認 |

### 品質管制

## 7. ERP 顧問訪談問題

### 製程 / BOM / 產能

1. 是否已有標準製程、產品用料、工時、機台與工具資料？
2. 自製與外包的判斷依據為何？是否會定期比較成本？
3. 經濟生產批量如何決定？是否由系統或人工維護？
4. 生產能量與工作負荷目前如何計算？是否有產能瓶頸資料？

### 計劃 / 排程 / 製造命令

1. 生產計劃來源是銷售預測、訂單、庫存水位或人工判斷？
2. 是否有總生產日程、製造日程、小日程或類似排程層級？
3. 製造命令如何開立、核准、派工與結案？
4. 每日生產進度如何回報？是否有生產日報表或電子紀錄？

### 倉儲 / 領退料 / 盤點

1. 原物料、半成品、製成品是否有統一編碼原則？
2. 領料與退料需要哪些單據與核准？
3. 安全存量、訂購點、請購點與經濟採購量是否已定義？
4. 盤點差異、呆廢料、廢料標售與報廢如何處理？

### 託外加工

1. 哪些製程會委外？委外判斷依據是產能、技術、成本或品質？
2. 協力廠商如何調查、試用、核准與定期考核？
3. 委外供料、餘廢料繳回、不良重工與扣款如何管控？
4. 委外加工付款是否需與加工合約、驗收與交貨資料勾稽？

### 品質 / 異常 / 重工

1. 原物料、製程、在製品、成品各階段檢驗標準是否不同？
2. 首件檢查、自主檢查、巡迴抽檢與複檢如何記錄？
3. 不良品、重工、停工、退料與製程調整如何核准？
4. 品質不良原因是否有統計分析與改善追蹤？

### 保養維修 / 安全衛生

1. 設備是否建立保養卡與維修履歷？
2. 年度與月份保養計劃如何排程與回報？
3. 故障請修、委託修造與保養材料使用如何控管？
4. 安全衛生教育訓練、環保與職業災害紀錄是否需納入系統管理？

### 成本 / 差異分析

1. 直接材料、直接人工、製造費用目前如何歸集？
2. 約當產量、製造費用分攤與多分攤 / 少分攤如何處理？
3. 是否使用標準成本？材料、人工、製造費用差異如何分析？
4. 成本差異是否有責任歸屬與改善追蹤？

## 9. 後續批次萃取格式建議

本次生產循環可沿用既有內控制度最小萃取格式，10 節結構、第 4 節 6 欄單據格式與第 6 節四類控制點分類均可使用。

但生產循環暴露出幾個後續可觀察的格式議題：

| 議題 | 目前處理方式 | 是否需立即升版 |
|---|---|---|
| 生產資料類型複雜 | 暫以第 4 節「單據」容納標準、計劃、命令、紀錄與表單 | 否 |
| 製令 / 工單 / 生產日報可能需系統細分 | 一律標示 `待系統確認` | 否 |
| 品質、設備、成本跨模組 | 先於訪談問題與待確認事項提示 | 否 |
| 領料、退料、在製品移轉原文抽取不完整 | 以 `[待確認]` 標示，不補造 | 否 |

建議先完成本草稿格式驗收，再決定是否更新制度範本總控。若後續薪工循環、研發循環或其他複雜循環也出現同類問題，再評估格式規則草稿是否升版為 v0.2。
