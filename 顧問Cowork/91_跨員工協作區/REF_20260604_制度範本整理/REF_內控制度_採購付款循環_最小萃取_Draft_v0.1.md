## 3. 採購 / 付款流程節點

```
預算
  → CP101 請購（請購單 → 庫存審核 → 主管核准）
      ↓
  ┌── CP102 採購（訪價 / 議價 / 比價 / 牌價 / 長期合約 / 招標）
  ├── CP103 進口（結匯 → 申請信用狀 → 在途 → 提貨 → 結匯還款）
      ↓
  CP104 驗收（廠商交貨 → 警衛登記 → 數量核對 → 檢驗 → 入庫 → 請款）
      ↓
  合格？
  ├── 否 → CP105 與合約不符（退換 / 扣款 / 減價 / 索賠）
  └── 是
      ↓
  CP106 付款（請款 → 財會核對 → 記帳 → 開支票 → 蓋印 → 領款）
      ↓
  CP107 差異分析（實際 vs 預算 / 標準成本差異分析、庫存呆廢料分析）
```

## 5. 角色 / 權責 / 簽核點

| 角色 / 單位 | 主要職責 | 主要簽核動作 |
|-------------|----------|--------------|
| 需求單位（製造 / 倉儲） | 發起請購、確認庫存、安排卸料與儲位 | 請購單核准 |
| 採購單位 | 詢比議價、決定採購方式、更新供應商資料、請款轉送 | 採購單 / 訂購單核准；轉送請款憑證 |
| 品管單位 | 材料取樣檢驗、開立材料檢驗報告單 | 檢驗合格 / 不合格判定 |
| 財會單位 | 審核請款、開立應付憑單、製作傳票、整理月結應付 | 請款條件審核；支付憑證審核 |
| 出納單位 | 開立支票、蓋印、安排付款 | 支票開立；抬頭劃線確認 |
| 印鑑保管人員 | 保管印鑑、審核憑證後蓋章 | 支票蓋章 |
| 警衛 | 廠商進出登記、核對送貨單與訂購單 | 進出廠登記 |
| 法務單位 | 合約簽訂前查核、違約案件法律途徑 | 合約查核 |
| 總經理 / 經營會 | 制度制修廢核准、特殊免劃線支票核准 | 最高核准 |

**主要簽核鏈（一般採購）：**
需求單位主管 → 採購單位主管 → 財會 → 出納 → 印鑑保管人 → 廠商領款

## 6. 主要控制點

### 請購階段

| 控制點 | 控制點類型 | ERP 落地確認 |
|--------|------------|--------------|
| 請購單需詳列廠商 / 規格 / 需用日期，倉儲先做庫存審核 | 單據控制 | 待系統確認 |
| 緊急採購須事後補請購單，需查明原因是否不可抗力 | 例外控制 | 待系統確認 |

### 採購階段

| 控制點 | 控制點類型 | ERP 落地確認 |
|--------|------------|--------------|
| 依金額與廠商數量決定訪價 / 議價 / 比價 / 招標 | 權責控制 | 待系統確認 |
| 三家以上廠商且金額達標時，以公開招標為原則 | 權責控制 | 待系統確認 |
| 重要合約簽訂前需法務查核；大量採購應保持兩家同時供料 | 權責控制 | 待系統確認 |

### 進口階段

| 控制點 | 控制點類型 | ERP 落地確認 |
|--------|------------|--------------|
| 預付款一律以支票（抬頭、劃線、禁止背書轉讓） | 單據控制 | 待系統確認 |
| 信用狀不得含可轉讓、可裝甲板等不利條款 | 單據控制 | 待系統確認 |
| 提貨時實收少於裝運數量需辦理索賠 | 例外控制 | 待系統確認 |

### 驗收階段

| 控制點 | 控制點類型 | ERP 落地確認 |
|--------|------------|--------------|
| 驗收需採購 + 驗收單位會同辦理 | 權責控制 | 待系統確認 |
| 發票品名 / 規格 / 數量 / 金額須與送貨單 / 驗收單相符 | 單據控制 | 待系統確認 |
| 不合格品不得入庫，需通知採購退回或扣款 | 例外控制 | 待系統確認 |

### 付款階段

| 控制點 | 控制點類型 | ERP 落地確認 |
|--------|------------|--------------|
| 支付一律開立抬頭劃線禁止背書轉讓支票（現金需核准） | 單據控制 | 待系統確認 |
| 已付款憑證需蓋「付訖」章，支票背後需領款人簽章，防重複付款 | 單據控制 | 待系統確認 |
| 出納不得以公司人員代領貨款（特殊情況需主管核准） | 例外控制 | 待系統確認 |

### 差異分析

| 控制點 | 控制點類型 | ERP 落地確認 |
|--------|------------|--------------|
| 實際採購量 / 金額與標準成本 / 預算定期比較 | 系統控制 | 待系統確認 |
| 人為因素造成差異需追查並懲戒 / 獎賞 | 例外控制 | 待系統確認 |
| 國外購置應利用避險工具控制匯率風險 | 權責控制 | 待系統確認 |

## 7. ERP 顧問訪談問題

> 以下問題適用於訪談客戶採購負責人 / 財會主管，了解客戶現況與 ERP 設定需求。

### 請購 / 採購
1. 請購單目前是紙本還是系統？主管是線上核准還是蓋章？
2. 採購方式（訪價 / 比價 / 招標）是否有金額門檻？現行門檻是多少？
3. 是否有核准的合格廠商清單（AVL）？在哪個系統管理？
4. 採購合約是否在 ERP 內管理，還是另有合約管理系統？

### 驗收
5. 驗收是否要求採購與品管共同會簽？目前流程是否有漏簽情形？
6. 材料檢驗報告是否整合進系統？不合格品如何在系統標記？
7. 計重材料的過磅數據如何進系統？手動輸入還是設備直連？

### 付款
8. 應付款項目前是系統自動沖帳還是人工核對？
9. 支票開立流程是否要求雙簽？印鑑幾枚？保管方式？
10. 付款是否有固定日期（如每月 5、20 日）？ERP 是否需設定自動到期提醒？
11. 目前是否有廠商要求非劃線支票的情形？如何處理？

### 差異分析
12. 採購成本差異報表目前在哪個系統出？頻率為何？
13. 國外採購是否有匯率避險操作？在 ERP 如何記帳？
14. 呆廢料分析現在的週期是多久？是否有自動觸發機制？
