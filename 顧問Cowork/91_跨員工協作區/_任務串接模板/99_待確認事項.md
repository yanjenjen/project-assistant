## 使用說明

1. 任一角色發現待確認事項時，直接新增一列至「待確認項目表」。
2. 解決後將該列移至「已解決項目歸檔區」，並填寫解決方式與日期。
3. 標示規則：資料不足請標示 `待補資料`；需客戶或顧問確認請標示 `需確認`。
4. 影響程度說明：高 = 影響上線或主流程；中 = 影響細節或規格；低 = 可延後釐清。
5. **必須進入 99 的項目類型**：凡影響主流程設計、系統設定、簽核規則、單據流、正式交付文件或上線節點的未確認事項，均必須彙整至本清單，不得僅記錄於個別模板中。
6. 欄位說明：
   - **確認對象**：誰負責推進確認（如：採購主管、IT 顧問、財務人員）
   - **確認方式**：會議 / 書面 / 口頭 / 系統驗證等
   - **預計釐清期限**：填寫具體日期或「待定」，要求有意識地填寫
   - **確認結果**：確認完成後填入結論
   - **後續動作**：確認後需執行的任務（如：更新 SOP 第 X 章、調整 Odoo 設定）
   - **影響檔案**：此待確認事項解決後需同步更新的檔案（如：01 / 02 / 03 / SOP）
