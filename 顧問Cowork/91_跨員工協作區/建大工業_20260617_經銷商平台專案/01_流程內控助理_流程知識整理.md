## 系統對應說明

> 系統整合架構：建大工業 ERP ↔ B2B 經銷商平台（艾創點開發）。核心開發原則（PPT 投影片28）：邏輯運算於 ERP 端完成、僅回傳最終結果；ERP 產單後須回傳正式訂單編號供雙向綁定；ERP 端檢核異常時直接擋單並回傳錯誤訊息。

| 流程節點 | ERP 模組 / 功能 | 對應說明 | 待確認項目 |
|----------|----------------|----------|------------|
| 產生正式訂單 | 產生正式訂單 API（本次討論新增） | B2B平台提供經銷商代號/成代/數量/金額；ERP 回傳正式訂單單號 | 屬「本次討論新增」，需確認目前開發狀態與是否已定案 |
| 報價單與價格檔同步 | 報價單與價格檔同步 API（本次討論新增） | B2B平台提供經銷商代號；ERP 回傳最新報價單號與單價 | 同上，需確認開發狀態 |
| 訂單資訊查詢 | 訂單資訊查詢 API | B2B平台提供ERP訂單單號；ERP回傳訂單狀態、已出貨數量、未出貨餘單量、出貨狀態、發票進度 | 需確認此 API 是否已完成開發（使用者表示整體已「開發中」） |
| 業績查詢 | 業績查詢 API | B2B平台提供經銷商代號＋查詢年月；ERP回傳結算完畢之業績總額（不傳細部明細） | 同上 |
| 獎金查詢 | 獎金查詢 API | B2B平台提供經銷商代號＋查詢年月；ERP回傳結算完畢之獎金數字 | 同上 |
| 對帳查詢 | 對帳查詢 API | B2B平台提供經銷商代號＋查詢年月；ERP回傳對帳金額、當期發票、當期折讓 | 同上 |
| 客戶主檔同步 | 不開發 API（排除項目） | Excel 一次性開帳匯入，後續經平台手動匯入維護 | 需確認開發中是否仍維持此排除範圍 |
| 產品主檔（成代）同步 | 不開發 API（排除項目） | 同上，Excel 一次性匯入與維護 | 同上 |
| 庫存查詢 | 不開發 API（排除項目） | 本次專案取消即時連動 | 同上 |
| 前置資料一致性 | 商品代號／客戶代號需 ERP 與 B2B 平台相同 | 為上述所有 API 串接之前製步驟 | 需確認雙方代號清單是否已完成核對 |

> 共計 6 支 ERP API（2 支本次新增＋4 支既有規劃），實際開發進度（已完成/開發中/未開始）逐支待確認，不可僅憑「整體開發中」籠統判斷。
